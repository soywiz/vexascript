# TypeScript native executable entrypoints

## Symptom

The native module graph already compiled `.ts` entrypoints to C++, but `vexa executable cli/cli.ts` failed before reaching it because the CLI path resolver only accepted `.vx` files. The JavaScript and direct C++ commands therefore supported TypeScript while the executable command rejected the same source.

## Resolution

Native executable path validation now accepts `.vx` and `.ts` and continues to reject unsupported source extensions before creating build artifacts. This removes the CLI-only gate and keeps native compilation on the existing shared TypeScript-aware module graph.

Release executables use `-O2`. Sanitizer builds retain `-O1`, and syntax-only validation retains `-O0`, because those modes optimize for diagnostics and compilation latency rather than release runtime performance.

The vendored mimalloc source archive is now the trimmed `mimalloc-3.4.3.zip` rather than the full `.tar.gz`. The first replacement attempt failed during CMake configuration because a source/include-only archive omitted `cmake/mimalloc-config-version.cmake` and `cmake/JoinPaths.cmake`. Keeping the existing unified CMake path was simpler and more portable than introducing a second direct-compiler path, so the ZIP now includes the complete three-file `cmake/` directory from the original archive. Upstream GitHub metadata, readmes, and security notes that CMake does not reference were removed; the source, headers, CMake input, package-config template, and license remain. A real native build configured mimalloc, built its override object, linked it, and ran successfully before the tracked tarball was removed. The ZIP is roughly 248 KB instead of 1.3 MB.

Single-file builds now report source/project/declaration loading, the semantic type-check, parse, analysis, emission, write, and total timings. This exposed the previously invisible cost in `build cli/cli.ts`: the external `pnpm exec tsc --noEmit` check dominates the roughly one-second native-CLI wall time and runs concurrently with VexaScript compilation, so phase durations do not sum to the total. Native executable builds report their graph phases and generation total before the long native step starts, announce `g++ -O2`, and report compile/link plus end-to-end time afterward. All compiler profiles use a shared `performance.now()` clock. The native emitter maps that call to a `std::chrono::steady_clock` implementation so the same compiler sources remain valid in browsers, Node.js, and self-hosted C++.

`build --emit cpp` originally retained a separate single-file emitter path while `cpp` and `executable` used the complete native module graph. Compiling `cli/cli.ts` through that isolated path failed on imported diagnostic types. `build --emit cpp` now delegates to the same graph function as `cpp`, and both commands always print project loading, declarations, graph analysis, C++ emission, type-check, write, and total timings.

## Regression coverage

Native path tests cover TypeScript entrypoints and unsupported extensions. Compiler-argument tests assert `-O2` for release and `-O1` for sanitizer builds, while the self-host test compiles the complete TypeScript CLI through `build --emit cpp` and validates the generated native translation unit.

The final self-host validation exposed an independent TypeScript emitter gap: ambient instance interfaces such as `Error` caused a real `extends Error` clause to be erased. That filtering is only valid for VexaScript's interface-conformance lowering. TypeScript emission now preserves syntactic `extends` clauses, with an isolated emitter regression test in addition to the full self-host roundtrip.

The first self-host reruns still appeared broken after that fix because an ignored `cli/cli.js`, previously generated by `vexa build cli/cli.ts`, shadowed the TypeScript source during dynamic-import bundling. A later full-suite run exposed the same class of collision with the intended extensionless native output `cli/cli`. Both local resolvers now prefer recognized source siblings before falling back to a direct extensionless file. This lets the executable remain beside `cli.ts` without changing what `import("./cli")` bundles.

Running the resulting native CLI exposed three later failures that were previously hidden by the first exception. First, C++ emission narrowed `string | undefined ?? string | undefined` to a native string and attempted to convert the remaining `undefined`; nullish coalescing now stays dynamically represented when both operands can still be undefined. Second, the JavaScript emitter constructed a `Program` with object spread, which correctly creates a structural record in native code but was later cast to the `Program` class; it now calls the real constructor. Third, optional native Array method calls discarded `?.` and could invoke `filter` with a null pointer; all native Array helpers now use the shared optional-receiver call path. Focused emitter tests cover the first and third cases, while native CLI self-host execution covers the full chain.
